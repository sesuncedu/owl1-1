Abstract

The entire community of OWL users, developers, and interested parties is invited to send their comments through the W3C mailing list. These issues will be logged on this page and will be discussed during the pending standardization process. Issues and bug reports are now managed through the public issues list.

  • http://lists.w3.org/Archives/Public/public-owl-dev/2006OctDec/0013.html
    RDF Mapping maps disjoint properties to owl:disjointWith, which may cause confusion (esp. with punning). (See issue 1.)
  • Annotation values are restricted to strings (that is, are more restricted than in OWL DL) (See issue 2.)
  • The current OWL 1.1 specification only allows one facet restriction per data range. This makes it cumbersome (and unintuitive) to specify ranges such as integer between 30 and 50. t should be possible to specify multiple facet restrictions for a data range. (See issue 10.)
  • Ontology consistency, concept satisfiability and subsumption are known to be EXPTIME-Complete for HORN-SHIQ (See issue 8.)
  • Entity annotations are currently allowed only in the entity declaration axioms. This makes it impossible to annotate an entity without declaring it. A potential correction to this problem would be to separate the entity declaration from the entity annotation axioms. (See issue 9.)
  • OWL 1.1 seems to provide for antisymmetric properties, but, by their semantics, the properties are asymmetric. (See issue 3.)
  • People are confused with the notation SPEi in section 7. It seems that it is not clear that SPE is yet another subproperty expression.
  • It should be made clear in the specification that languageTag is formatted according to RFC 3066. (See issue 18.)
